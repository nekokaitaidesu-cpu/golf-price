# ⛳ 本命スナイパー（新着割安出品のLINE通知）

夜の本命抽出は「残り物 = 訳あり」に勝てないので、こちらは**速度**で勝負する。
人気加速中の機種（popularity.jsonから毎朝自動選定・上位20機種）を15〜30分おきに巡回し、
**出品直後の割安・状態良好・シャフト付き**だけをLINEにプッシュする。

## 通知される条件（すべて満たす時だけ）
- ウォッチ機種の新着出品（出品から2時間以内・初見のみ）
- 価格が実売中央値の **35〜72%**（下限未満は部品・別物の経験則）
- ヘッド単品・部品・ジャンクでない（既存フィルタ流用）

状態ランクでは除外しない（傷ありも通知に載る。通知文の状態表示＋写真で判断する方針）。
絞りたくなったら `COND_MAX = 4` に。

※写真の傷までは見られない。**通知が来ても現物写真の確認は必須**（2026-07-07の教訓）。

## セットアップ（1回だけ）

### 1. LINEトークンをこのPCに設定
TP07ハンターと同じLINE公式アカウントを使い回せる。
[LINE Developersコンソール](https://developers.line.biz/console/) の
Messaging APIチャネルから「チャネルアクセストークン」と「あなたのユーザーID」を確認して:

```
setx LINE_CHANNEL_TOKEN "（トークン）"
setx LINE_USER_ID "（Uから始まるID）"
```

（GitHubのSecretsに登録済みの値は読み出せないので、コンソールから再取得）

### 2. 動作テスト
```
python notify_honmei.py --dry-run
```
初回は現在の出品を既知登録するだけ（シード）。通知は2回目以降の新着から。

### 3. タスクスケジューラ登録（15分おきの例）
```
schtasks /Create /SC MINUTE /MO 15 /TN "honmei-notify" /TR "C:\Users\User\Claude\golf-price\notify_honmei.bat"
```
解除は `schtasks /Delete /TN "honmei-notify" /F`。ログは `notify.log`。

## チューニング（notify_honmei.py 冒頭の定数）
| 定数 | 既定 | 意味 |
|---|---|---|
| `RATIO_MAX` | 0.72 | 中央値の何%以下で「割安」とするか。鳴らなすぎたら0.8に |
| `COND_MAX` | 4 | 許容する状態ランク（4=やや傷あり まで） |
| `MAX_MODELS` | 20 | 監視機種数。増やすとスキャンが長くなる |
| `LOOKBACK_SEC` | 2時間 | 見る新着の範囲。実行間隔より長くしておく |
